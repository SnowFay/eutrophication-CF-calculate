# 0.1° 网格 FF/CF 精度提升路线（建议稿）

本文档假设“当前链路在工程口径上正确（B→C→D→E 可复现且单位无错）”，讨论如何提升结果精度与科学可信度。

精度提升分两类：
- A. 不改变总体框架（仍以 0.1° 网格输出，仍 single-downstream 递推）：投入小、收益大，建议优先做。
- B. 改变内部计算单元（reach/子流域优先）：投入较大，但能解决结构性误差。

---

## 0) 从学术研究角度：当前工作主要问题与整改建议（可直接写进论文/答辩）

### 0.1 研究对象与指标定义不够“可比”（概念层）
**问题**
- 文献常给“全国平均因子”，但其通常是**在特定排放情景/权重规则下的区域加权平均**；若你用网格等权/面积等权/仅在中国掩膜内做统计，会天然得到不同的全国值，导致“偏高/偏低”的结论难以站住。
- FF、CF、以及污染物形态（TP vs DIP/DOP/PP）与受纳介质（freshwater）若在表述上不够严谨，评审会认为你在“对不齐指标”。

**整改（最小可行）**
- 明确并固定一套“对标口径”：
  - 你输出的是 **FF（day）** 还是 **CF（=FF×EF）**；
  - 污染物定义（例如 TP-P 还是 PO4-P）；
  - 受纳环境边界（河流/湖库是否都算 freshwater）。
- 提供至少两套全国汇总：
  1) 网格等权/面积权重（描述“空间分布”）；
  2) 排放/负荷加权（描述“典型排放的平均归趋”），并声明权重数据来源。

**验收标准**
- 在结果章节能用一句话说清：**“本研究的全国均值是在 X 权重、Y 情景、Z 指标定义下得到的”**，并能与至少一篇文献做到同口径对比。

### 0.2 缺少“独立验证链路”（有效性层）
**问题**
- 目前更多是内部一致性（工程正确）而非外部有效性（科学正确）。仅凭“看起来合理”不足以支撑发表或答辩。

**整改（分三档，优先做 A→B）**
- A. 文献对标（最快）：选 3–5 个可复现的对标量（全国均值、典型流域出口量级、湖库驻留时间量级），把对标口径写清。
- B. 观测/第三方数据交叉验证（中等）：
  - 选若干大流域（长江/黄河/珠江等）出口 Q 与公开统计对比；
  - 选若干大湖/水库的体积/驻留时间与公开资料比对。
- C. case study 复现（最强）：复现 1 个已有研究区的 FF/停留时间结论（哪怕只复现量级与空间格局）。

**验收标准**
- 至少给出 1 张“对标表”或“对标图”：列出对标对象、口径、你的结果、参考值、误差范围与原因解释。

### 0.3 不确定性与敏感性尚未量化（可辩护性层）
**问题**
- 参数（如 VF、用水去向系数）、输入（湖库体积、河道体积估计、Q 口径）、以及拓扑简化（单下游）都会系统性影响 FF；不量化不确定性，结论难以辩护。

**整改**
- 做“有限预算”的敏感性分析：对 5–8 个关键因子做 ±30% 或分段扰动，输出全国均值/分位数变化与空间差异。
- 把拓扑不确定性显式化（见 2.1），把“结果不可靠区域”从最终展示中标注出来（而不是藏在均值里）。

**验收标准**
- 结果章节包含：关键输入/参数的敏感性排序（Top 5）与“对全国均值影响范围”。

### 0.4 可复现性与可审计性仍可加强（研究工程化层）
**问题**
- 评审/合作方关心：每个变量的来源、单位、处理规则、以及版本（数据版本/脚本版本/阈值常数）是否可追溯。

**整改**
- 在 NetCDF 写入更完整的全局属性：数据源版本、关键常数（如最小体积阈值）、生成时间、脚本路径/commit hash（若有）。
- 给每个产物配一页“数据字典/变量表”（可放 materials/ 或补充材料）。

**验收标准**
- 任意一个结果变量都能追溯到：输入数据集→处理脚本→关键假设/阈值→单位。

### 0.5 目前“归趋（FF）”与“效应（EF）/表征（CF）”闭合度不足（方法学完整性层）
**问题**
- 若当前主要产出仍是 FF（时间量），而 EF 尚未按同一污染物定义与受纳环境完成区域化，则你在 LCIA 意义上的 CF 仍是不完整的。
- 即使计算了 CF，也需要清楚写明：EF 的来源、单位、以及与 FF 的时间单位如何抵消（避免“年/天换算导致 CF 改变”的误解）。

**整改**
- 明确论文/报告的交付边界：
  - 若主贡献是“高分辨率 FF 数据底座”，则把 EF/CF 放在展望；
  - 若要交付 CF，则补齐 EF，并做单位闭合检查（CF 对 day↔yr 变换应保持不变，前提是 EF 同步变换）。

**验收标准**
- 方法学章节出现一段“单位闭合/维度分析”，并能解释 FF、EF、CF 的单位与换算。

### 0.6 学术贡献与范围边界需要更清晰（写作与发表策略层）
**问题**
- 若仅描述“把多个数据集映射到 0.1° 并跑通公式”，容易被评价为工程集成而非研究贡献。

**整改**
- 把贡献写成可检验的研究产出（至少满足其中 2 条）：
  1) 可审计的全国 0.1° FF/CF 产品（含不确定性/诊断层）；
  2) 拓扑不确定性显式化与质量控制框架（可迁移到其他区域/分辨率）；
  3) 与文献/第三方数据的系统对标与误差归因（解释“为何偏低/偏高”）；
  4) 空间聚合与加权规则的标准化（省/市/流域可比）。

**验收标准**
- 摘要/结论能用 2–3 句概括“新在哪里、比现有工作强在哪里、可复用在哪里”。

## 1) 当前误差来源分解（帮助确定优先级）

### 1.1 拓扑误差（结构性）
- 单下游指针强制化：三角洲/分汊/湖区多出口被简化为一个出口。
- 网格切割：河道与分水岭被规则网格切割。
- 河网驱动拓扑不等同于 DEM-D8：下游可能非 8 邻格；局部跳跃或断裂会放大 FF 误差。

### 1.2 水量口径误差（驱动层）
- `q_best_cms` 是“HydroRIVERS 主河段 Q 与 runoff 累积 Q 的混合口径”。不同来源在局地可能不一致。
- 时间口径不完全一致：Q / runoff / 用水是长期平均还是特定年份，需要明确。

### 1.3 过程参数与模型结构误差（过程层）
- `k_ret` 的简化：VF×水体面积 + kriv(Q) 分段，忽略水深/温度/水体类型差异。
- `k_use` 用水去向系数（f_WTA/f_DITW/f_soil）是强假设，且与地区产业结构相关。
- 稳态一阶假设忽略季节性。

## 2) 优先级最高：不改变框架的提升（建议先做）

### 2.1 拓扑不确定性显式化（最高优先级）
目标：让 `next_grid_id` 不再是黑盒，并能识别“哪里不可信”。

建议新增诊断层/表：
- `n_outlet_candidates`：本网格可能的出格河段数量
- `main_reach_id`：作为出口的主河段标识
- `outlet_score_main` 与 `outlet_score_2nd`：主/次候选得分
- `outlet_score_gap`：不确定性强弱（gap 小表示不稳定）
- `topology_flag`：例如 stuck/cycle_break/multi_outlet 等

验收标准：
- `cycle_break_flag` 区域更集中且可解释
- “疑难格子”空间分布不随机（主要在分汊/湖区/数据稀疏区）

### 2.2 Q 口径一致性改进（高优先级）
目标：减少 HydroRIVERS Q 与 runoff 累积 Q 的系统性偏差。

具体做法（由轻到重）：
1) 先做诊断：输出 `q_hydrorivers_main_cms / q_accum_runoff_cms` 的地图与分区统计。
2) 引入规则：当两者差异超过阈值（例如 10× 或 0.1×），标记 `q_flag` 并采用更稳健的混合策略。
3) 分区校准：按大流域对 runoff→Q 做比例校正，使出口 Q 更接近公开统计量级。

验收标准：
- `q_ratio` 的极端尾部显著收敛
- 关键流域出口的 Q 量级更合理

### 2.3 水体体积/面积质量控制（高优先级）
目标：避免单个异常湖库主导 τ 与 FF 的长尾。

建议：
- 计算等效平均水深：`depth_eq_m = (V_waterbody_km3 * 1e9) / (A_waterbody_km2 * 1e6)`
- 对超阈值（例如 >300m 或你设定的更保守阈值）做 flag，而不是硬删。
- 对 flagged 格子在结果展示中单独标注，并在敏感性分析中测试“截断/替代”对 FF 分布的影响。

验收标准：
- Top100 极端格子可被解释为真实大湖/大库或被 flag 为异常

### 2.4 参数区域化（中高优先级）
目标：减少全国统一常数带来的结构性偏差。

候选项：
- VF 按气候带/温度带分区（至少南北分区）
- kriv(Q) 分段保持不变，但可按河道类型/坡度（若有）做修正
- 用水去向系数按产业结构或城乡分区

验收标准：
- 改进后对标验证（文献范围/空间格局）更一致
- 参数扰动敏感性降低（更稳健）

## 3) 中期升级：改变内部计算单元（投入大但更“水文一致”）

### 3.1 reach/子流域单元上路由，再汇总回 0.1°
思路：
- 在 HydroRIVERS reach 网络或 HydroBASINS 子流域网络上计算 τ 与递推（天然拓扑一致）
- 再把结果按映射（reach→grid overlap 或 basin→grid overlap）汇总成 0.1° 输出

优点：
- 减少规则网格切割导致的拓扑误差
- 对分汊/多出口更容易处理（可保留多下游并在网格汇总时折算）

成本：
- 需要构建 reach/basin 级别的“无损映射+网络”与新一套输出逻辑

验收标准：
- 与当前网格版本差异主要集中在分汊/湖区
- 对标与敏感性结果更稳健

## 4) 对外沟通：把“精度提升”变成可写进论文的方法学

建议写法：
- 主文：给出核心框架（single-downstream、稳态一阶、0.1° 输出）
- 补充材料：
  - 拓扑诊断与不确定性地图
  - Q 一致性诊断与校准策略
  - 参数敏感性分析（±30%）

这样评审能清楚看到：你不仅给出结果，还给出“不确定性边界”。
