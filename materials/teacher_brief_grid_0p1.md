# 给老师汇报用：0.1° 网格淡水 FF/CF（Freshwater 子模块）计算逻辑与下一步计划

> 目的：用尽可能少的概念，把“我现在到底在算什么、怎么一步步算出来、每个关键参数代表什么物理过程、当前做法合理吗、下一步怎么验证与提精度”讲清楚，便于老师快速参与讨论。

---

## 0. 我现在在算什么（一句话 + 边界）

- **我现在算的是“淡水系统内部（freshwater）的 fate 子模块”**：假设物质已经在某个 0.1° 网格单元 *进入淡水系统*（河道/湖库/汇流网络），计算它沿水路到出口的平均停留时间与传输效应。
- **我现在没有算**：从空气/土壤到进入淡水的那段路径（例如大气沉降到地表、土壤吸附/淋溶、地表径流把 P 带入河网等）。这些将作为后续“多介质耦合”的上游模块。

因此，本阶段输出的 `FF_freshwater(i)`（day）应理解为：

- “若物质在网格 i 进入淡水系统（entry to freshwater），从 i 沿水路到出口的 fate factor”。

---

## 1. 为什么每个网格都可能有 FF？（用一句物理直觉解释）

- 这套框架是 **grid-based 汇流单元**：每个格子都代表一个“产流/汇流单元节点”，即使格子里没有明显的主河道线，它仍会通过径流产生水量并汇入下游。
- 所以：**“只有河网路径上有数据”**是 reach-based（只在河段上定义状态）模型的直觉；而我们是 grid-based（每格节点）模型，很多变量自然会在大部分格子上有值。

对“河网格子”和“非河网格子”数值来源的澄清（避免冲突理解）：

- 河网数据覆盖到的格子（有 HydroRIVERS 主河段 overlap）：
  - 该格子的 `q_best_cms` 往往直接来自 HydroRIVERS 的多年平均流量（更像“河道观测/整合产品”）。
  - 该格子的 `v_river_km3`、河长等也更直接依赖河段 overlap（线要素在格子内的长度）。
- 非河网格子（没有可靠主河段）：
  - 该格子的 `q_best_cms` 会回退到 `q_accum_runoff_cms`（由径流深 + 汇流累积得到）。
  - 物理含义是“这个格子作为产流/汇流单元，对下游贡献的水量”，并不要求格子内一定有被显式描绘的主河道线。
- 是否冲突：
  - **逻辑上不冲突**：`q_best` 是一个“优先河道流量、缺失则用产流汇流”的统一口径，用于让全网格都能驱动后续 k 与 FF。
  - **但这是不确定性来源之一**：河道流量与面源汇流累积在局部可能不一致，会导致 `q_best` 在空间上出现“切换带”。我们用 `q_source`（流量来源标记）来显式记录，并建议做 `q_hydrorivers_main_cms` vs `q_accum_runoff_cms` 的一致性诊断图来评估影响。

如果老师希望看到“只在河道网络上的 FF”，我们可以额外输出一个 river-mask 版本（例如只保留 `river_length_km_in_cell>0` 的格子），但那将对应更窄的排放情景（直接入河）。

## 3. 整体流程（B → C → D → E）

下面按“论文方法学”的顺序给出：每一步输入什么、输出什么、以及关键参数的物理意义。

### Step B：把各种数据无损映射到 0.1° 网格，并构建单下游拓扑

- **无损映射**：对每个数据集输出 mapping parquet。
  - “每行是 `grid_id × feature_id` 的相交记录”指：
    - `grid_id`：一个 0.1° 网格单元；
    - `feature_id`：原始数据集里的一个要素（例如某条河段、某个湖库面、某个流域多边形）；
    - “相交记录”指该要素的几何（线/面）与该网格单元的几何发生了空间相交（overlay/intersection），因此这个要素对这个格子有贡献。
  - “保留 overlap（面积/长度/比例）”指把相交部分的几何量也存下来：
    - **线要素（河段）**：`overlap_length`（该河段落在该格子内的长度）。
    - **面要素（湖库/流域）**：`overlap_area`（该面要素落在该格子内的面积）。
    - **比例（overlap_frac）**：相交量占该要素总量的比例（例如某湖库有 30% 面积在这个格子里）。
  - 为什么要保留：
    - 这是“可审计/可回溯”的关键：任何网格值都能追溯到哪些要素贡献了多少。
    - 也是做加权聚合的依据：例如流域径流深按 `overlap_area` 加权；河段属性按 `overlap_length` 或“主河段”规则选取；湖库体积可按 `overlap_frac` 分摊到多个格子。
- **单下游拓扑**：为每个格子定义 `next_grid_id`（下游格子）。
  - 物理意义：代表这个格子的水（以及携带的物质）主要沿哪个方向流出并进入下游格子。
  - 说明：这是“河网驱动的单下游指针”，不强制是 DEM-D8 的 8 邻格。
  - 与 DEM-D8（8 邻格下泄）的差异与优缺点（讨论要点）：
    - DEM-D8：下游必须是 8 邻格中坡降最大的方向。
      - 优点：完全由地形决定、局地规则简单、拓扑天然是栅格邻接（便于水量守恒与数值实现）。
      - 缺点：在平原/洼地/人工河道/分汊区容易把真实河道“算歪”；对 DEM 质量敏感；无法直接利用 HydroRIVERS 的已知河网连通性。
    - 河网驱动单下游指针：下游由河网要素的连通关系主导，允许“跳到非 8 邻格”。
      - 优点：更贴近已有河网产品的连通结构，主干流方向通常更可信；对平原/人工河道更稳健。
      - 缺点：在分汊/三角洲会被强制单出口（结构性简化）；若河网产品本身局部有错，会直接影响拓扑；并且“非 8 邻格跳跃”需要额外诊断以避免不合理断裂。

输出：`grid_topology_china_0p1.nc`（包含 `hydrorivers_next_grid_id` 等）。

### Step C：生成 D/E 需要的权威水文与几何输入（把口径统一）

目标：给每个格子准备好后续速率计算需要的 **水量 Q**、**体积 V**、**水体面积 A** 等。

1) 径流深 `runoff_m_per_year`（m/yr）
- 物理意义：单位面积每年产生多少径流深度（可以理解为“从降水中转化为地表/河网径流的水量产出强度”的长期平均）。
- “面源水量”这里指：相对于“某条河段的线源流量”，径流深是对整个地表/流域面都定义的水量生成量。
- 数据来源：在当前实现中主要来自 BasinATLAS 这类**流域面数据**（每个流域多边形都有一个径流深属性），再通过 mapping 的 `overlap_area` 加权映射到每个网格。
- 覆盖范围：只要网格落在这些流域面多边形覆盖范围内，就会有数值；因此中国陆地区域往往会“几乎每格都有 runoff”。
- 用途：用于计算本地产流量，并在无河段流量时提供流量 fallback。

2) 本地产流量 `q_local_runoff_cms`（m³/s）
- 计算：`runoff_m_per_year × cell_area / seconds_per_year`。
- 物理意义：格子自身贡献到河网/下游的水量。

3) 沿 `next_grid_id` 累积得到 `q_accum_runoff_cms`
- 物理意义：到达该格子的总汇流水量（上游贡献 + 本地贡献）。

4) 河网主河段流量 `q_hydrorivers_main_cms`（m³/s）
- 从 HydroRIVERS 映射表里选“覆盖长度最长”的河段作为主河段。
- 物理意义：如果该格子确实有主要河段，HydroRIVERS 提供的多年平均流量通常比 runoff 累积更接近河道观测/整合产品。

5) 最终用于后续的流量口径 `q_best_cms`
- 规则：优先 HydroRIVERS 主河段流量；若缺失则用 `q_accum_runoff_cms`。
- 物理意义：把“河道观测/整合产品”与“面源产流路由”合并成一个可用的流量驱动。

6) 水体体积与面积
- `waterbody_area_km2_in_cell`：格子内湖库水面面积（km²）。
- `v_waterbody_km3_in_cell`：格子内湖库体积（km³）。
- `v_river_km3`：格子内河道水体体积（km³），来自河宽/水深/河长的几何估计。
  - 其中河宽/水深目前用 Wollheim 等（2006）的幂律关系由流量 Q 推算得到；河长来自 HydroRIVERS 在格子内的 overlap_length 汇总。
- `v_tot_km3 = v_river_km3 + v_waterbody_km3_in_cell`：淡水总水体体积。

重要口径：HydroLAKES 体积字段 `Vol_total/Vol_res` 的单位是 mcm（百万立方米），需要换算为 km³（1 mcm = 0.001 km³）。

输出：`grid_hydro_geom_china_0p1.nc`。

### Step D：把水文与几何转换成过程速率（day⁻¹）与停留时间（day）

目标：在每个格子上构建一阶过程模型。

核心中间量与关键参数（含来源与数值）：

- `q_best_cms`（m³/s）：本阶段流量的“权威入口”。
  - 来源：Stage C 输出 `grid_hydro_geom_china_0p1.nc`。
  - 生成规则（在 Stage C 已定义）：优先 HydroRIVERS 主河段多年平均流量；缺失则使用 runoff 沿 `next_grid_id` 的累积流量。
- `Q`（km³/day）：速率计算使用的体积流量。
  - 计算：`Q = q_best_cms * 86400 / 1e9`。
  - 单位换算常数：`86400 s/day`，`1 km³ = 1e9 m³`。
- `V_tot`（km³）：淡水系统的“控制体体积”。
  - 来源：Stage C 输出 `v_tot_km3`（`v_tot_km3 = v_river_km3 + v_waterbody_km3_in_cell`）。
  - 物理意义：把格子内河道水体与湖库水体体积相加，作为该格子水体停留与过程速率的分母。
- `V_riv`（km³）：河道水体体积。
  - 来源：Stage C 输出 `v_river_km3`。
- `A_wb`（km²）：湖库水面面积。
  - 来源：Stage C 输出 `waterbody_area_km2_in_cell`（由 HydroLAKES 面要素按 overlap 聚合）。

文献参数（直接写死在 Stage D 脚本中）：

- `v_f`（km/day）：水体吸收速度（uptake velocity）。
  - 数值：`v_f = 3.8e-5 km/day`。
  - 来源：Helmes 等（Online Resource）引用 Alexander (2004) 的设定；在脚本中常量名为 `VF_KM_PER_DAY`。
- `k_riv`（day⁻¹）：河道滞留/去除强度（随流量等级分段）。
  - 计算需要 `q_best_cms` 的年尺度换算：`Q_yr = q_best_cms * 31557600 / 1e9`（km³/yr；其中 `31557600 s/yr = 365.25 day/yr`）。
  - 分段常数（来源：Helmes 等 Online Resource 的 kriv 分段；脚本内常量 `KRIV_*`）：
    - 若 `Q_yr < 0.0882 km³/yr`：`k_riv = 71.2 yr⁻¹`
    - 若 `0.0882 ≤ Q_yr < 0.4473 km³/yr`：`k_riv = 25.0 yr⁻¹`
    - 若 `Q_yr ≥ 0.4473 km³/yr`：`k_riv = 4.4 yr⁻¹`
  - 转为 day⁻¹：`k_riv_day = k_riv_yr / 365.25`。

数值稳定参数：

- `V_tot_min`（km³）：体积下限，用于避免 `Q/V_tot` 在极小体积格子爆炸。
  - 数值：`V_tot_min = 1e-9 km³`（等价 1 m³）。
  - 规则：当 `V_tot < V_tot_min` 时，该格子的 `k_adv/k_ret/k_use/tau` 输出为 NaN（脚本常量 `V_TOT_MIN_KM3_FOR_RATES`）。

(1) 平流/输运速率 `k_adv`（day⁻¹）

- 公式：`k_adv = Q / V_tot`
- 物理意义：单位时间内被水流“带走/更新”的比例；流量越大、体积越小，输运越快。

(2) 滞留/去除相关的速率 `k_ret`（day⁻¹）

- 简化结构：`k_ret = (V_riv * k_riv + v_f * A_wb) / V_tot`
- 物理意义：
  - `k_riv`：河道内部的有效滞留/去除强度（用文献分段常数表示，并随流量等级变化）。
  - `v_f * A_wb`：水体表面的“等效吸收/沉降”项，其中
    - `v_f`（km/day）是吸收速度（uptake velocity），表示单位时间内有效被去除的水柱厚度；
    - `A_wb`（km²）是水体面积，面积越大，表面过程越强。

(3) 人类用水相关速率 `k_use`（day⁻¹）

- 结构：`k_use = f_WTA * (1 - f_DITW) * k_adv * (1 - f_soil)`
- 各参数物理意义：
  - `f_WTA`（0–500）：用水需求与可供水量的比值（Water-to-Availability）。越大表示“人类取用”相对越强。
  - `f_DITW`（0–1）：用水中进入“污水处理路径”的比例（domestic+manufacturing+electric 的占比）。
  - `f_soil`（0–1）：表示从源到水体的土壤相关截留/延缓的经验比例（由径流深 R 推得）。在本阶段它作为一个经验调制项参与 `k_use`。

这三个系数在当前实现里的“来源/计算方式/取值规则”（对应 Stage D 脚本 `compute_k_params_grid_0p1.py`）：

- `f_WTA`：来自“用水需求 / 可供水量”的比值（逐格计算，不是外部直接给定常数）。
  - 用水需求（m³/day）：来自 HSWUD 数据（`hswud_0p1_2010_2020_mean_clean_m3_per_day.parquet`）按网格汇总后的 `q_total_m3_per_day`。
  - 可供水量（m³/day）：来自本格子流量 `q_best_cms` 换算 `q_main_m3_day = q_best_cms * 86400`。
  - 计算：`f_WTA = q_total_m3_day / q_main_m3_day`（仅当 `q_main_m3_day > 0`）。
  - 特殊情况与截断：
    - 若 `q_total_m3_day <= 0`：设为 0（无需求）。
    - 若 `q_total_m3_day > 0` 但 `q_main_m3_day` 过小导致比值非有限：设为 500。
    - 最终 `clip` 到 [0, 500]（500 是人为设定的上限，用来避免数值爆炸）。

- `f_DITW`：来自“用水中进入生活/工业/电力等可被视作污水处理路径的占比”（逐格计算）。
  - 分子（m³/day）：`q_dom_m3_day + q_manu_m3_day + q_ele_m3_day`（同样来自 HSWUD，并按网格汇总）。
  - 分母（m³/day）：`q_total_m3_day`。
  - 计算：当 `q_total_m3_day > 0` 时 `f_DITW = ditw_num / q_total_m3_day`；否则设为 0。
  - 取值规则：最终 `clip` 到 [0, 1]。
  - '（后续应该考虑纳入不同行业污水处理的参数）

- `f_soil`：来自文献经验公式（由径流深决定），用于近似“土壤/陆面过程导致的延缓/截留”强度。
  - 驱动变量：`R = runoff_m_per_year`（m/yr），来源 Stage C（主要由 BasinATLAS 面数据映射得到）。
  - 计算步骤（脚本实现引用 Helmes Online Resource 的公式结构；在代码中以 `f_dip`/`f_dop` 表达）：
    - `f_dip = 0.29 / (1 + (R / 0.85)^(-2))`
    - `f_dop = 0.01 * R^(0.95)`
    - `f_soil = clip(f_dip + f_dop, 0, 1)`
  - 说明：这一步本质上是把“更湿润/径流更强的区域”映射到不同的经验截留/转化强度；属于结构性假设，后续可以通过敏感性分析与参数区域化来评估与改进。

(4) 停留时间 `tau`（day）

- 公式：`tau = 1 / (k_adv + k_ret + k_use)`
- 物理意义：在该格子内（作为控制体）的平均停留时间。

数值稳定规则：当 `V_tot` 极小会导致 `Q/V` 爆炸，因此对 `V_tot` 设置最小阈值（等价于 1 m³）来掩膜这些格子的速率输出。

输出：`grid_k_params_china_0p1.nc`。

### Step E：沿单下游指针递推得到 Fate Factor（FF）

定义传输分数：

- `f_transfer = k_adv / (k_adv + k_ret + k_use)`
- 物理意义：从该格子“成功向下游传输”的比例（剩余部分在本格子被滞留/去除/用水影响）。

递推公式（单下游网络）：

- `FF(i) = tau(i) + f_transfer(i) * FF(next(i))`
- 边界：若无下游（出界/出口），则 `FF(i) = tau(i)`。
- 同时输出：
  - `export_frac_to_outlet`：沿路径的 `f_transfer` 连乘（到出口的总体传输分数）。
  - `steps_to_outlet`：到出口的步数。
  - `cycle_break_flag`：若拓扑存在环，则在局部断边以保证递推可解，并记录。

输出：`grid_ff_cf_china_0p1.nc`。

## 4. 当前做法是否合理？（我希望跟老师确认的关键点）

我认为当前阶段应采用“任意网格进入淡水”的定义（而不是只在河道格子上定义），理由是：

- 目标是最终的多介质 CF：空气/土壤模块将把排放“分配到进入淡水的位置”，因此淡水子模块需要对任意进入点都能给出 `FF_freshwater(i)`。
- 如果只用“河道格子 FF”（river-mask），会强迫把面源/土壤淋溶/大气沉降等过程硬映射到最近河道格子，容易引入结构性偏差。

需要老师帮助判断/讨论的假设点：

1) grid-based 汇流单元是否适合作为“进入淡水”的空间承载单元？
2) `q_best`（HydroRIVERS Q 与 runoff 累积 Q 的混合）是否需要进一步的“一致性校准”？
3) `k_ret` 与 `k_use` 的简化结构是否满足当前阶段的“可解释 + 可验证”？

## 5. 下一步如何验证（可执行清单，建议先做最小集合）

### 5.1 必做：机制/守恒验证（排除实现与单位错误）

- 范围检查：`f_transfer ∈ [0,1]`；`k_* ≥ 0`；`tau > 0`（或 NaN 有明确原因）。
- 递推恒等式抽样复算：随机抽样格子 i，检查 `FF(i) ≈ tau(i) + f(i) * FF(next(i))`。
- 拓扑健全性：`cycle_break_flag` 数量与空间位置是否集中在可解释区域（分汊/湖区/内流区）。

### 5.2 推荐：外部对标（建立数量级可信度）

- Q 对标：选长江/黄河/珠江等出口附近格子，对比 `q_best_cms` 与公开多年平均流量数量级。
- FF 对标：汇总全国 `ff_day` 分位数（p50/p90/p99/max），对照文献典型范围与空间格局描述。

### 5.3 强烈推荐：敏感性与稳健性（最能说服答辩/审稿）

- 关键常数 ±30%：对 `v_f`、`k_riv`（分段常数）、以及 `f_WTA` cap 做扰动，比较 FF 分布与空间格局变化。
- 拓扑替换对比：若存在替代指针（如 riveratlas），对比两套 FF 差异，定位“拓扑不确定区域”。

## 6. 下一步如何提升精度（按优先级排序 + 具体操作）

### 6.1 最高优先级：把拓扑不确定性显式化（让 next 不再是黑盒）

- 新增诊断层：每格出口候选数、主/次候选得分差（gap）、主河段 id。
- 产出：一张“不确定性地图”，让老师能直接指出“这里网络可能不可靠”。

### 6.2 高优先级：改进 Q 口径一致性

- 先做诊断图：`q_hydrorivers_main_cms / q_accum_runoff_cms` 的地图与分区统计。
- 若差异系统性偏大：按大流域对 runoff→Q 做比例校准，使出口 Q 更接近公开统计量级。

### 6.3 高优先级：水体体积/面积的 QA（避免异常湖库主导长尾）

- 计算等效水深：`depth_eq = V/A`（m）。
- 对超阈值格子做 flag（不一定删除），并在敏感性分析里测试“截断/替代”对 FF 的影响。

### 6.4 中高优先级：参数区域化

- `v_f`、用水去向等按气候带/产业结构分区，而不是全国统一常数。

### 6.5 中期升级（投入大）：reach/子流域单元上先路由，再汇总回 0.1°

- 优点：拓扑更水文一致，分汊/多出口更易处理。
- 成本：需要 reach/basin 级网络与新的汇总逻辑。
